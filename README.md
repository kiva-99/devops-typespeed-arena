# ⌨️ TypeSpeed Arena

> **DevOps-платформа для тренировки слепой печати с демонстрацией современных практик инфраструктуры**

🚧 **Статус проекта:** Архитектура  на утверждении | Локальный запуск работает  
📅 **Дата обновления:** 2026-03-25  
👤 **Автор:** Иванов Кирилл Константинович  
📦 **Репозиторий:** [GitHub](https://github.com/kiva-99/devops-typespeed-arena)

## 🎯 Цель проекта

Разработка веб-приложения для тренировки скорости печати с **полной демонстрацией DevOps-практик**:

| Приоритет | Фокус | Описание |
|-----------|-------|----------|
| 🔴 **Первично** | **Инфраструктура** | Контейнеризация, CI/CD, IaC, оркестрация, мониторинг |
| 🟡 **Вторично** | **Бизнес-логика** | Простое приложение (тренажёр печати) для демонстрации инфраструктуры |

## 🚀 Быстрый старт (локальная разработка)

### Требования

| Компонент | Минимальные требования | Рекомендованные |
|-----------|----------------------|-----------------|
| **Docker Desktop** | v20.10+ | v29.1.3 (актуальная) |
| **Оперативная память** | 4 ГБ | 8+ ГБ |
| **Свободное место** | 2 ГБ | 5+ ГБ |
| **Доступ к интернету** | Требуется для pull образов | Стабильный |

### Запуск в одну команду

```powershell
# 1. Клонируем репозиторий
git clone https://github.com/kiva-99/devops-typespeed-arena.git
cd devops-typespeed-arena

# 2. Переходим в папку с Docker Compose
cd deploy/local

# 3. Запускаем все сервисы (фоновый режим)
docker-compose up -d

# 4. Проверяем статус (все 3 контейнера должны быть Up)
docker-compose ps

# 5. Открываем в браузере
start http://localhost

### Проверка работы

# Тестовый запрос к API (должен вернуть JSON со списком текстов)
curl http://localhost:5001/api/texts

# Просмотр логов бэкенда (для отладки)
docker-compose logs -f backend

# Просмотр логов всех сервисов
docker-compose logs -f

### Остановка

# Остановить контейнеры (данные сохранятся в томах)
docker-compose down

# Остановить и удалить тома (полный сброс БД)
docker-compose down -v
```

## ☁️ Стратегия деплоя: Гибридная модель

Проект может быть развёрнут как на локальной машине разработчика одной командой docker-compose up, так и в облачной инфраструктуре с автоматическим масштабированием через Terraform. Это демонстрирует принцип портативности и Infrastructure as Code.

## 🛡️ Высокая доступность и отказоустойчивость

### Database Tier (PostgreSQL HA)
```
 
┌─────────────────────────────────────┐
│         Приложение (Flask)          │
│   DATABASE_URL: postgresql://...    │
└────────────────┬────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────┐
│         Patroni (HA Manager)        │
│  • Отслеживает состояние кластера   │
│  • Автоматически выбирает новый     │
│    master при сбое текущего         │
│  • Интеграция с etcd для консенсуса │
└───────┬─────────────────┬───────────┘
        │                 │
        ▼                 ▼
┌───────────────┐ ┌───────────────┐
│ PostgreSQL #1 │ │ PostgreSQL #2 │
│   Primary     │ │   Replica     │
│   (write)     │ │   (read-only) │
└───────────────┘ └───────────────┘
```
### Application Tier (Backend HA)

-Минимум 2 реплики Flask-приложения за Nginx/Ingress
-Health checks: /health endpoint для проверки готовности
-Graceful shutdown: корректная остановка при деплое
-Resource limits: CPU/RAM лимиты в Docker/K8s для предотвращения "шумных соседей"

### 📊 Мониторинг и логирование (план)

# Метрики (Prometheus)
1.Всего запросов к API
2.Время обработки запроса
3.Ошибки 5xx
4.Активные подключения к БД
5.Средняя скорость печати

# Дашборды (Grafana)
1.Infrastructure: CPU/RAM/Disk всех нод
2.Application: RPS, latency, error rate
3.Business: Активные пользователи, среднее WPM, топ текстов
4.Database: Репликация лаг, количество подключений, размер БД

# Логи (Loki)
1.Сбор логов со всех контейнеров через Promtail
2.Парсинг JSON-логов Flask для структурированного поиска
3.Алерты на паттерны: ERROR, Exception, Connection refused
